## 1. 产品概述

### 1.1 产品定位
铁腕是一款基于艾森豪威尔矩阵的智能任务管理工具，帮助用户快速记录待办事项，科学规划任务优先级。

### 1.2 目标用户
需要管理日常任务、提升效率的个人用户。

### 1.3 MVP目标
- 快速记录待办任务
- 支持桌面端和移动端使用
- 账号云端登录系统 (Supabase Auth)
- *双端数据同步 (Supabase Database)*

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 艾森豪威尔矩阵
任务按优先级分为四个象限：

| 象限 | 名称 | 说明 | 优先级值 |
|------|------|------|---------|
| 第一象限 | 立即做 | 重要且紧急 | 1 |
| 第二象限 | 计划做 | 重要但不紧急 | 2 |
| 第三象限 | 委托做 | 不重要但紧急 | 3 |
| 第四象限 | 不做 | 不重要且不紧急 | 4 |

#### 2.1.2 任务管理
- **添加任务**: 输入内容，选择象限
- **完成任务**: 点击复选框标记完成（触发粒子彩带特效），随后任务淡出并从矩阵中消失。
- **历史已完成列表**: 页面左上角提供菜单按钮，点击从左侧划出抽屉栏，按时间维度**互斥分类**展示（三个 Tab 之间无重叠）：
  - **今日**: 今天（00:00 至今）完成的任务
  - **本周**: 本周一 00:00 至昨日 23:59 完成的任务（不含今日，与今日 Tab 互斥）
  - **历史**: 本周一之前完成的所有任务
- **删除任务**: 可以在抽屉栏或未完成矩阵中删除任务。

#### 2.1.4 统计与成就系统 (新增)
- **侧边栏入口**: 页面左上角提供菜单按钮，点击后划出左侧边栏。
- **统计列表**（三个 Tab 互斥分类，无重叠）:
  - **今日已完成**: 展示今天（00:00至今）完成的归档和未归档任务。
  - **本周已完成**: 展示本周一至昨日完成的任务（不含今日，与今日 Tab 互斥）。
  - **历史已完成**: 展示本周一之前完成的所有任务。

#### 2.1.3 用户系统
- **注册**: 邮箱 + 密码
- **登录**: 邮箱 + 密码
- **退出登录**: 清除本地登录状态

## 3. 技术方案

### 3.1 架构图

```text
┌────────────────────────────────────────────────────────────┐
│                          用户端                             │
│   ┌─────────────┐              ┌─────────────┐              │
│   │  电脑浏览器  │              │  手机浏览器  │              │
│   │  (PWA网页)  │              │  (PWA网页)  │              │
│   └──────┬──────┘              └──────┬──────┘              │
│          │                            │                    │
│          └────────────┬───────────────┘                    │
│                       │                                     │
│             ┌─────────▼──────────┐                         │
│             │   前端 (Frontend)   │                        │
│             │  HTML + CSS + JS   │                        │
│             └─────────┬──────────┘                         │
└───────────────────────┼────────────────────────────────────┘
                        │
┌───────────────────────▼────────────────────────────────────┐
│                      数据存储 (Storage)                     │
│                  当前状态：Supabase (PostgreSQL)            │
│                  未来状态：支持离线缓存与多端实时同步       │
└────────────────────────────────────────────────────────────┘
```

### 3.2 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | HTML5 + CSS3 + ES6 | 原生前端技术实现 |
| 存储 | Supabase | 提供 Auth 和 Database，实现云端多端同步 |
| 部署 | Vercel (Static) | 采用 Vercel 静态托管部署，发布 `viewer01` 目录 |

### 3.3 数据模型

#### 用户集合 (Supabase Auth)
依托 Supabase 官方 `auth.users` 表管理：
- `id`: 用户唯一标识 (UUID)
- `email`: 邮箱 (用于登录)
- `password`: 密码 (Supabase加密存储)
- `created_at`: 注册时间

#### 任务集合 (tasks)
保存在 Supabase `tasks` 表中，字段：

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `id` | UUID | 是 | 任务ID (主键) |
| `content` | Text | 是 | 任务内容 |
| `completed` | Boolean | 是 | 是否完成，默认false |
| `quadrant` | Text | 是 | 象限 q1/q2/q3/q4 |
| `user_id` | UUID | 是 | 关联用户ID (外键 `auth.users.id`) |
| `archived` | Boolean | 否 | 是否已归档（默认false）。归档后不在四象限显示，仅在统计中显示 |
| `completed_at`| Timestamp | 否 | 任务被标记为完成的时间（用于分类统计） |
| `created_at` | Timestamp | 自动 | 创建时间 |
| `updated_at` | Timestamp | 自动 | 更新时间 |

#### 用户配置集合 (user_configs)
保存在 Supabase `user_configs` 表中，字段：
- `id`: UUID (主键)
- `user_id`: UUID (外键 `auth.users.id`，唯一)
- `ai_provider`: Text (AI 提供商，如 'deepseek')
- `ai_api_key`: Text (AI 密钥)
- `ai_model`: Text (选中的模型)
- `custom_api_base`: Text
- `custom_model`: Text
- `ui_quadrant`: Text (最后一次选中的象限)
- `updated_at`: Timestamp

## 4. 页面结构

### 4.1 登录/注册页
- 登录表单：邮箱、密码、登录按钮
- 注册表单：邮箱、密码、确认密码、注册按钮
- 切换链接："还没有账号？立即注册" / "已有账号？立即登录"

### 4.2 任务管理页（主页面）
- 顶部：左侧菜单按钮(展开历史记录)、Logo、用户信息、退出按钮
- 主体：艾森豪威尔矩阵四象限展示（仅展示未完成任务）
- 侧边栏抽屉：分为三个Tab/区域展示当日、当周、全部的历史已完成任务
- 底部：任务输入栏

## 5. 用户操作流程

### 5.1 首次使用
1. 打开网页
2. 点击"注册账号"
3. 输入邮箱、密码
4. 注册成功，自动登录
5. 进入任务管理页

### 5.2 日常使用
1. 打开网页
2. 输入邮箱、密码登录
3. 查看任务列表
4. 添加/完成/删除任务
5. 数据自动保存到本地浏览器

## 7. 部署方案

### 7.1 本地开发
- 运行 `npm run dev` 启动本地静态服务器
- 数据保存在浏览器 `localStorage` 中

### 7.2 云端部署 (Supabase)
- 前端通过 Supabase JS SDK 直接连接云端数据库。
- 后端数据保存在 Supabase 的 PostgreSQL 数据库中。
- 用户认证通过 Supabase Auth 进行。
