## 1. 产品概述

### 1.1 产品定位
铁腕是一款基于艾森豪威尔矩阵的智能任务管理工具，帮助用户快速记录待办事项，科学规划任务优先级。

### 1.2 目标用户
需要管理日常任务、提升效率的个人用户。

### 1.3 MVP目标
- 快速记录待办任务
- 支持桌面端和移动端使用
- 双端数据同步
- 账号登录系统

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 艾森豪威尔矩阵
任务按优先级分为四个象限：

| 象限 | 名称 | 说明 | 优先级值 |
|------|------|------|---------|
| 第一象限 | 立即做 | 重要且紧急 | 1 |
| 第二象限 | 计划做 | 重要但不紧急 | 2 |
| 第三象限 | 委托做 | 不重要但紧急 | 3 |
| 第四象限 | 不做 | 不重要且不紧急 | 4 |

#### 2.1.2 任务管理
- **添加任务**: 输入内容，选择象限
- **完成任务**: 点击复选框标记完成/未完成
- **删除任务**: 删除已完成的任务
- **查看任务**: 按象限分组展示

#### 2.1.3 用户系统
- **注册**: 用户名 + 密码
- **登录**: 用户名 + 密码
- **退出登录**: 清除本地登录状态

### 3.2 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | HTML5 + CSS3 + ES6 | 基于现有UI改造 |
| 后端 | LeanCloud (BaaS) | 国内云服务，中文支持 |
| 数据库 | LeanCloud 数据存储 | NoSQL，自动同步 |
| 部署 | LeanCloud 静态托管 | 免费，国内访问快 |

### 3.3 数据模型

#### 用户表 (_User)
LeanCloud 内置用户表，字段：
- `username`: 用户名（唯一）
- `password`: 密码（加密存储）
- `createdAt`: 注册时间
- `updatedAt`: 更新时间

#### 任务表 (Task)
自定义数据表，字段：

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `content` | String | 是 | 任务内容 |
| `completed` | Boolean | 是 | 是否完成，默认false |
| `quadrant` | Number | 是 | 象限 1-4 |
| `user` | Pointer | 是 | 关联用户 |
| `createdAt` | Date | 自动 | 创建时间 |
| `updatedAt` | Date | 自动 | 更新时间 |

## 5. 用户操作流程

### 5.1 首次使用
1. 打开网页
2. 点击"注册账号"
3. 输入用户名、密码
4. 注册成功，自动登录
5. 进入任务管理页

### 5.2 日常使用
1. 打开网页
2. 输入用户名、密码登录
3. 查看任务列表
4. 添加/完成/删除任务
5. 数据自动同步到云端

### 5.3 换设备使用
1. 在新设备打开网页
2. 用相同账号登录
3. 任务数据自动同步显示

## 6. 非功能需求

### 6.1 性能要求
- 页面加载时间 < 3秒
- 任务操作响应 < 1秒
- 支持同时在线用户：免费额度内无限制

### 6.2 兼容性
- 桌面端：Chrome, Firefox, Safari, Edge 最新版
- 移动端：iOS Safari, Android Chrome

### 6.3 安全要求
- 密码加密存储（LeanCloud自动处理）
- 用户只能访问自己的任务数据
- 使用 HTTPS 传输

## 9. 部署方案

### 9.1 LeanCloud 静态托管配置
1. 在 LeanCloud 控制台创建应用
2. 进入"存储" → 创建 Class 名为 `Task`
3. 进入"设置" → "安全中心" → 配置安全域名
4. 进入"部署" → "静态网站托管" → 上传前端文件

### 9.2 环境变量配置
前端代码需要配置以下参数（由用户注册后提供）：
- `APP_ID`: LeanCloud 应用 ID
- `APP_KEY`: LeanCloud 应用 Key
- `SERVER_URL`: 服务器地址（国内版一般为 https://xxx.lc-cn-n1-shared.com）
