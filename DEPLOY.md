# 铁腕任务管理工具 - 部署指南

## 前置要求

1. **Node.js** (v14+)
2. **CloudBase CLI** - 安装命令：
   ```bash
   npm install -g @cloudbase/cli
   ```

3. **腾讯云账号** - 已完成实名认证

## 部署步骤

### 1. 配置 CloudBase 环境

运行配置脚本：
```bash
npm run setup
```

或手动执行：
```bash
node cloudbase-setup.js
```

此脚本会：
- 登录 CloudBase
- 创建 tasks 数据库集合
- 配置安全规则
- 开启静态网站托管

### 2. 本地测试

启动本地开发服务器：
```bash
npm run dev
```

访问 http://localhost:8080 查看效果

### 3. 部署到线上

一键部署：
```bash
npm run deploy
```

部署完成后，CloudBase 会提供访问域名。

## 手动配置（备用方案）

如果自动脚本执行失败，可以手动配置：

### 1. 登录控制台
访问 https://console.cloud.tencent.com/tcb

### 2. 创建数据库集合
- 进入「数据库」模块
- 点击「新建集合」
- 集合名称：`tasks`

### 3. 配置安全规则
在数据库页面，点击「权限设置」，添加规则：
```json
{
  "read": "doc._openid == auth.openid",
  "write": "doc._openid == auth.openid"
}
```

### 4. 开启静态托管
- 进入「静态网站托管」模块
- 点击「开启」

### 5. 部署文件
- 在静态托管页面，点击「上传文件」
- 上传 `viewer01` 目录下的所有文件

## 验证部署

部署完成后，访问提供的域名，验证：
1. 可以打开登录页面
2. 可以注册新账号
3. 可以登录
4. 可以添加任务
5. 数据可以同步

## 故障排查

### 问题：无法登录 CloudBase CLI
**解决**：运行 `tcb login` 手动登录

### 问题：部署失败
**解决**：
1. 检查环境 ID 是否正确
2. 检查是否有部署权限
3. 尝试手动上传文件

### 问题：数据无法同步
**解决**：
1. 检查数据库集合是否创建
2. 检查安全规则是否配置
3. 检查网络连接
